# SEC-014: Service security boundary

> **SYNTHETIC DEMO POLICY** — This is synthetic content for a bounded public
> demonstration. It is not organizational policy or a production decision.

## Intent

Protect RelayEU service boundaries against transport, identity, replay, and
observability failures.

## Applicability

Applies to RelayEU workloads where `platform` equals `azure`.

## Required properties

<a id="sec-014-tls"></a>
### `sec-014-tls` — TLS

Require TLS for service and client transport.

<a id="sec-014-entra-status-api"></a>
### `sec-014-entra-status-api` — Entra status API

Expose the status API through an Entra-protected boundary.

<a id="sec-014-es256-jwks"></a>
### `sec-014-es256-jwks` — ES256 and JWKS

Validate ES256 tokens using the issuer JWKS.

<a id="sec-014-replay"></a>
### `sec-014-replay` — Replay resistance

Reject replayed requests using bounded freshness and replay detection.

<a id="sec-014-safe-logging"></a>
### `sec-014-safe-logging` — Safe logging

Do not log tokens, secrets, or confidential payload values.

## Expected evidence

Provide transport settings, Entra boundary configuration, token-validation
design, replay-control design, and representative redacted logs.

## Exceptions and waivers

This demo policy grants no exception or waiver. Record a request for owner
review; only the simulated policy-owner process may publish a future change.
