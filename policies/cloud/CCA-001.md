# CCA-001: Azure platform baseline

> **SYNTHETIC DEMO POLICY** — This is synthetic content for a bounded public
> demonstration. It is not organizational policy or a production decision.

## Intent

Keep RelayEU cloud workloads on an approved, manageable Azure baseline.

## Applicability

Applies to RelayEU workloads where `platform` equals `azure`.

## Required properties

<a id="cca-001-approved-services"></a>
### `cca-001-approved-services` — Approved services

Use approved Azure services for the workload topology.

<a id="cca-001-approved-regions"></a>
### `cca-001-approved-regions` — Approved regions

Deploy only in approved EU Azure regions.

<a id="cca-001-managed-identity"></a>
### `cca-001-managed-identity` — Managed identity

Use managed identities rather than embedded workload credentials.

<a id="cca-001-private-data"></a>
### `cca-001-private-data` — Private data paths

Keep confidential data on private network paths.

<a id="cca-001-bicep"></a>
### `cca-001-bicep` — Infrastructure as code

Declare Azure infrastructure with Bicep.

## Expected evidence

Provide the Azure architecture, selected regions and services, managed identity
configuration, private endpoint design, and reviewed Bicep source.

## Exceptions and waivers

This demo policy grants no exception or waiver. Record a request for owner
review; only the simulated policy-owner process may publish a future change.
